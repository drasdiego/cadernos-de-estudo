01O que é Banco de Dadosestudado em 26 jun 2026
Conceito Um banco de dados é uma coleção organizada de dados armazenados eletronicamente em um sistema de computador. Ele permite gerenciar, buscar e manipular essas informações de forma eficiente.

Tabelas: a estrutura básica

Dentro de um banco, os dados ficam em tabelas, organizadas em linhas e colunas:

  • Linha (registro): representa uma ocorrência completa — por exemplo, um cliente específico.
  • Coluna (campo / atributo): representa uma característica do dado — por exemplo, "nome" ou "CPF".
Analogia Pense numa planilha do Excel. Cada aba é uma tabela, cada linha é um cadastro e cada coluna é uma informação daquele cadastro. Um banco de dados é isso, porém preparado para muitos dados e muitos acessos ao mesmo tempo, com regras de integridade.
Na prática Em outras palavras: banco de dados = onde a informação mora, de forma organizada.
O que isso significa: toda consulta SQL que você escrever depois vai agir sobre essas linhas e colunas.
02O que é SQLestudado em 26 jun 2026
Conceito SQL (Structured Query Language) é a "Linguagem Estruturada de Consulta". É o idioma que usamos para conversar com o banco de dados: criar tabelas, inserir, alterar e consultar dados.
  • Criada no início dos anos 70, pela IBM.
  • Linguagem de altíssimo nível: bem próxima da linguagem humana (inglês). Comandos como SELECT, INSERT e DELETE já dizem o que fazem.
  • É uma linguagem declarativa: você descreve o resultado que quer, não o passo a passo de como obtê-lo.
  • Ao longo do tempo surgiram novos dialetos, derivando e evoluindo o SQL.
Dica — declarativo vs. imperativo Declarativo (SQL): "quero os clientes de SP". Você não diz como percorrer a tabela; o banco resolve.
Imperativo (ex.: um for em Python): você escreve cada passo da busca.
Por que importa: isso explica por que SQL é tão enxuto. Você foca no "o quê", e o motor do banco cuida do "como".
Dica — o que é "dialeto" Um dialeto é uma versão do SQL adaptada por um fabricante. Exemplos: MySQL, PostgreSQL, SQL Server (T-SQL), Oracle (PL/SQL). O núcleo é o mesmo; mudam alguns detalhes de sintaxe.

Conceito O SQL é a linguagem usada em praticamente todos os SGBDs do mercado.
Dica — o que é SGBD SGBD = Sistema de Gerenciamento de Banco de Dados (em inglês, DBMS). É o software que administra o banco: guarda os dados, garante segurança e executa seus comandos SQL. Exemplos: MySQL, PostgreSQL, Oracle, SQL Server.
Atenção Mesmo com a padronização, cada fabricante tem suas características próprias. Um comando pode funcionar no MySQL e exigir pequeno ajuste no SQL Server. Por isso, sempre confira a documentação do banco que você está usando.
04As 5 sublinguagens do SQLestudado em 26 jun 2026

O SQL não é um bloco único. Ele é formado por cinco conjuntos de comandos, cada um com uma finalidade. Entender essa divisão organiza todo o resto da matéria.

Mapa mental — o "guarda-chuva" SQL
SiglaNomePara quê serve
DDLData Definition LanguageDefinir a estrutura (criar/alterar/excluir tabelas).
DMLData Manipulation LanguageManipular os dados (inserir, alterar, deletar registros).
DQLData Query LanguageConsultar dados (SELECT).
DCLData Control LanguageControlar acessos e permissões.
DTLData Transaction LanguageGerenciar transações (confirmar ou desfazer).
Na prática Quando estudar qualquer comando novo, primeiro pergunte: "de qual sublinguagem ele faz parte?". Isso já diz o propósito dele e evita confusão.
05DDL — Linguagem de Definição de Dadosestudado em 26 jun 2026
Conceito A DDL cuida da estrutura do banco. Ela não mexe nos dados em si, e sim no "esqueleto": as tabelas e suas colunas.
ComandoAção
CREATECria uma estrutura (ex.: uma tabela).
ALTERAltera uma estrutura existente.
DROPExclui uma estrutura.

CREATE TABLE — criar uma tabela

Código SQL 
-- Forma geral:
CREATE TABLE nomeTabela (campo tipo atributo, ... );

-- Exemplo: tabela de clientes
CREATE TABLE CLIENTE (
  codigo     int          NOT NULL PRIMARY KEY,
  nome       varchar(30)  NOT NULL,
  cpf        int          UNIQUE,
  uf         char(2)      DEFAULT null,
  nascimento date         DEFAULT '0000-00-00',
  telefone   char(8)      DEFAULT null,
  sexo       char(1)
);
Lendo a sintaxe Cada coluna segue o padrão nome · tipo · atributos. Exemplo: nome varchar(30) NOT NULL = coluna "nome", texto de até 30 caracteres, que não pode ficar vazia.

ALTER TABLE — alterar uma tabela

Código SQL 
-- Incluir uma nova coluna:
ALTER TABLE CLIENTE ADD email VARCHAR(20);
ALTER TABLE CLIENTE ADD fax VARCHAR(18) AFTER uf;

-- Eliminar uma coluna existente:
ALTER TABLE CLIENTE DROP fax;

-- Alterar o tipo de dado de uma coluna:
ALTER TABLE CLIENTE MODIFY email VARCHAR(40);

DROP TABLE — excluir uma tabela

Código SQL 
DROP TABLE [nome da tabela];
Atenção — cuidado com DROP e ALTER...DROP DROP TABLE apaga a tabela inteira, com todos os dados, e não tem "lixeira". ALTER TABLE ... DROP coluna apaga só a coluna. São ações destrutivas: confirme o ambiente antes de rodar.
06DML — Linguagem de Manipulação de Dadosestudado em 26 jun 2026
Conceito A DML trabalha com o conteúdo das tabelas: inserir, alterar e apagar registros (linhas). A estrutura já existe; aqui mexemos nos dados.
ComandoAção
INSERTInsere dados (novas linhas).
UPDATEAltera dados existentes.
DELETEDeleta dados.

INSERT INTO — inserir registros

Código SQL 
INSERT INTO TABELA VALUES (valores);

-- Exemplo:
INSERT INTO TABELA VALUES
(123, 'Allan Rocker', '12345', 'SP',
 '1234-12-12', '123456', 'M', 'allan@email.com');

UPDATE — atualizar registros

Código SQL 
UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;

DELETE — apagar registros

Código SQL 
DELETE FROM table_name WHERE condition;
Atenção — o perigo de esquecer o WHERE Em UPDATE e DELETE, a cláusula WHERE define quais linhas serão afetadas. Sem WHERE, o comando atinge TODAS as linhas da tabela. Um DELETE FROM CLIENTE; sem WHERE apaga todos os clientes.
Na prática Hábito de segurança: antes de rodar um UPDATE/DELETE, escreva primeiro um SELECT com o mesmo WHERE para ver exatamente quais linhas serão afetadas.
07DQL — Linguagem de Consulta de Dadosestudado em 26 jun 2026
Conceito A DQL é dedicada a consultar dados. O comando central é o SELECT, o mais usado no dia a dia de quem trabalha com dados.

O SELECT também permite:

  • Ordenação de dados (ex.: por nome, por data).
  • Agrupamento de dados (juntar por categoria).
  • Funções aritméticas (somar, contar, calcular médias).
  • Filtros de seleção (escolher só o que interessa).

SELECT — consultar

Código SQL 
SELECT *
FROM TABELA
WHERE nome = 'Allan';
Lendo a consulta SELECT * = "traga todas as colunas". FROM TABELA = "da tabela TABELA". WHERE nome = 'Allan' = "apenas as linhas onde a coluna nome for igual a Allan".
Dica de leitura: mentalmente, o banco lê primeiro o FROM, depois o WHERE e por último o SELECT.
Na prática Como você está em transição para Dados, o SELECT será sua ferramenta mais frequente. Vale dominar bem WHERE, ORDER BY e GROUP BY nas próximas aulas.
08DCL — Linguagem de Controle de Dadosestudado em 26 jun 2026
Conceito A DCL controla quem pode fazer o quê no banco. É a parte de permissões e segurança de acesso.
ComandoAção
GRANTHabilita (concede) acesso a dados e operações.
REVOKERevoga (retira) acesso a dados e operações.
Analogia — chaves de acesso Pense num prédio. GRANT é entregar a chave de uma sala a alguém. REVOKE é recolher essa chave. O DCL define quem entra em quais "salas" (tabelas) e o que pode fazer lá dentro.
Na prática No seu contexto de serviços financeiros, DCL é o que garante que cada pessoa veja apenas os dados que tem permissão. É um pilar de segurança e governança de dados.
09DTL — Linguagem de Transação de Dadosestudado em 26 jun 2026
Conceito A DTL gerencia transações. Uma transação é um conjunto de operações tratado como uma unidade única: ou tudo dá certo e é confirmado, ou nada é aplicado.
ComandoAção
START TRANSACTIONInicia a transação.
COMMITConcretiza (confirma) a transação.
ROLLBACKAnula (desfaz) a transação.
Analogia — transferência bancária Numa transferência, dois passos precisam acontecer juntos: debitar de uma conta e creditar em outra. Se o sistema cair no meio, não pode debitar sem creditar.
COMMIT = confirma os dois passos juntos. ROLLBACK = se algo falhou, desfaz tudo e volta ao estado inicial. Ninguém perde dinheiro.
Na prática Em outras palavras: transação garante o "tudo ou nada". Por que importa: é o que dá confiabilidade a sistemas financeiros, evitando estados pela metade.
10Atributos de colunasestudado em 26 jun 2026
Conceito Atributos são regras e propriedades que você aplica a uma coluna no momento de criá-la. Eles controlam o que aquela coluna aceita ou como se comporta.
AtributoO que faz
NULL / NOT NULLPermite ou não valores nulos na coluna. NULL é diferente de vazio: nulo = "ausência de valor / desconhecido".
SIGNED / UNSIGNEDPermite ou não números negativos. UNSIGNED = só positivos (útil para idade, quantidade).
AUTO_INCREMENTGera sequências automáticas (contadores). Cada novo registro recebe o próximo número.
ZEROFILLPreenche o número completando com zeros à esquerda (ex.: 7 vira 007).
Dica — NULL não é zero nem espaço NULL significa "não há informação". É diferente de 0 (um número) e de '' (texto vazio). Exemplo: um cliente sem telefone cadastrado tem telefone NULL, não telefone "0".
Na prática AUTO_INCREMENT conecta com o próximo tópico: é a forma mais comum de gerar uma chave primária automática (a chave surrogada).
11Chaves: PK, FK, UNIQUE e Surrogadaestudado em 26 jun 2026
Conceito — o que são chaves Chaves são atributos (ou conjuntos de atributos) que identificam de forma única um registro numa tabela. São essenciais para a integridade dos dados: previnem duplicatas e permitem criar relacionamentos entre tabelas. Sem chaves, seria impossível gerenciar dados de forma eficaz.
Chave Primária (Primary Key — PK) Atributo (ou conjunto) que identifica de forma única cada registro da tabela. Garante que não existam dois registros com a mesma PK. Só pode haver uma por tabela e é sempre NOT NULL.
Exemplo: na tabela Clientes, o ID do cliente é a PK — cada cliente tem um identificador único.
Chave Estrangeira (Foreign Key — FK) Campo de uma tabela que cria um vínculo com a PK de outra tabela. Mantém a integridade referencial: o valor da FK precisa sempre corresponder a uma PK válida na tabela relacionada.
Exemplo: em Pedidos, o campo id_cliente é FK e aponta para a PK de Clientes. Não existe pedido de um cliente que não existe.
PK e FK trabalhando juntas A PK identifica a linha dentro da própria tabela. A FK é a referência dessa PK em outra tabela, criando a ligação. É esse par que torna possível relacionar tabelas (próximo tema).
Conceito — restrições UNIQUE e PRIMARY KEY Ambas asseguram que os valores de uma ou mais colunas sejam únicos em cada linha da tabela. Servem para não ter duplicidade (ex.: dois clientes com o mesmo CPF).
RestriçãoCaracterística
PRIMARY KEYIdentificador principal da linha. Só pode haver uma por tabela. É única e obrigatoriamente NOT NULL.
UNIQUEGarante unicidade, mas pode haver várias na mesma tabela. Usada em colunas que também devem ser únicas (ex.: e-mail, CPF).
Regra — relação com NOT NULL Colunas definidas como PRIMARY KEY ou UNIQUE geralmente também precisam ser NOT NULL. O atributo NOT NULL impede valores nulos e é obrigatório em colunas com restrição PRIMARY KEY.
Conceito — Chave Surrogada (Surrogate Key) Uma chave surrogada é uma chave artificial, geralmente numérica, criada apenas para servir de chave primária. Ela substitui uma chave natural que seria complexa ou não ideal.
Dica — surrogada vs. natural Chave natural: um dado real do mundo que identifica a linha, como o CPF.
Chave surrogada: um número sem significado de negócio, criado só para identificar (ex.: codigo com AUTO_INCREMENT).
Por que usar surrogada: é simples, estável e rápida. Se a regra de negócio mudar (ex.: passar a aceitar clientes estrangeiros sem CPF), a chave artificial continua funcionando.
12Relacionamentos entre tabelasestudado em 26 jun 2026
Conceito Relacionamento é a ligação entre tabelas, feita por chaves: a Primary Key (PK) de uma tabela aparece em outra como Foreign Key (FK). A "cardinalidade" diz quantas linhas de um lado se ligam ao outro.
Dica — o que é Foreign Key (FK) A chave estrangeira é uma coluna que aponta para a chave primária de outra tabela. É o que conecta, por exemplo, uma Compra ao Cliente que a realizou. Ela garante a integridade referencial (não existe compra de um cliente que não existe).
Diagrama — os três tipos de cardinalidade
Na prática 1:1 → cada um, um só (Pessoa ↔ CPF). 1:N → um para muitos (Cliente → Compras). N:M → muitos para muitos (Compras ↔ Produtos).
Atenção: bancos relacionais não implementam N:M diretamente. Cria-se uma tabela intermediária (associativa) que transforma o N:M em dois relacionamentos 1:N.
13Modelos de Dadosestudado em 26 jun 2026
Conceito Modelos de dados são formas de organizar e estruturar os dados num banco. A escolha depende do objetivo: operação do dia a dia ou análise.
Mapa mental — quando usar cada modelo

Detalhamento

Modelo Relacional Organiza dados em tabelas inter-relacionadas por chaves primárias e estrangeiras. Ideal para operações transacionais: inserir, atualizar, excluir e consultar com garantia de integridade referencial. É o modelo da maior parte dos sistemas do dia a dia (ex.: o cadastro de clientes do banco).
Modelo Dimensional Usado principalmente em data warehouses e sistemas de BI (Business Intelligence). Organiza os dados em duas categorias: fatos (dados quantitativos, como vendas) e dimensões (contexto/descritores, como tempo, produto ou localização).
Star Schema (Esquema Estrela) É um tipo de modelo dimensional: uma tabela de fatos central conectada a várias tabelas de dimensão, formando o desenho de uma estrela. É simples e eficiente para consultas, facilitando análises rápidas e agregações de dados.
Na prática — conexão com sua transição para Dados Relacional = mundo da operação (registrar o que acontece). Dimensional / Star Schema = mundo da análise (entender o que aconteceu). Na sua migração para Dados e BI, você sairá do relacional em direção ao dimensional.
14Filtros e a cláusula WHEREestudado em 26 jun 2026
Conceito — o que é o WHERE O WHERE ("onde") filtra os resultados de uma consulta: define quais linhas aparecem. O banco testa a condição linha por linha — se for verdadeira, a linha aparece; se falsa, é ignorada.
Por que usar: foco (ver só o que importa) e performance (processa menos dados, busca mais rápida).

Tabela de exemplo — PESSOAS

IDNOMEIDADEFILHOS
1Eliana200
2Miguel393
3Raquel362
4João221
5Marta482

Operadores de comparação

O que são Fazem uma pergunta aos dados e devolvem um valor booleano (verdadeiro ou falso). Booleano = tipo de dado com apenas dois estados possíveis.
OperadorSignificado
=Igual a
!= ou <>Diferente de
>Maior que
<Menor que
>=Maior ou igual
<=Menor ou igual
Código SQL 
-- Idade maior ou igual a 39:
SELECT * FROM PESSOAS WHERE IDADE >= 39;
-- Resultado: Miguel (39) e Marta (48)

-- Diferente de zero filhos:
SELECT * FROM PESSOAS WHERE FILHOS <> 0;
-- Resultado: Miguel, Raquel, João, Marta

Álgebra booleana: AND, OR, NOT

Combinando condições AND (E): só retorna se todas as condições forem verdadeiras. Analogia: receita que exige "ovo E leite" — falta um, não dá.
OR (OU): retorna se pelo menos uma for verdadeira. Analogia: "suco OU refrigerante".
NOT (NÃO): inverte — traz o oposto da condição.
Código SQL 
SELECT * FROM PESSOAS
WHERE IDADE IN (39, 48)
  AND FILHOS > 2;
-- Só Miguel: tem 39 anos E mais de 2 filhos (3).
-- Marta sai: tem 48, mas só 2 filhos.

Intervalo, lista, texto e nulos

OperadorO que faz
BETWEENValor dentro de um intervalo (inclui as pontas). Ex.: IDADE BETWEEN 20 AND 23.
INValor que coincide com algum item de uma lista. Ex.: UF IN ('SP','RJ','MG').
LIKEBusca padrões em texto. % = vários caracteres; _ = um caractere.
IS NULL / IS NOT NULLCampo vazio (sem informação) ou preenchido.
DISTINCTRemove duplicatas; retorna só valores exclusivos.
Código SQL 
-- Nomes que começam com M:
SELECT * FROM PESSOAS WHERE NOME LIKE 'M%';
-- Miguel e Marta

-- Idade entre 20 e 23 (inclui as pontas):
SELECT * FROM PESSOAS WHERE IDADE BETWEEN 20 AND 23;
-- Eliana (20) e João (22)
Atenção — LIKE e wildcards % representa zero ou muitos caracteres; _ representa exatamente um. 'M%' = começa com M. '%a' = termina com a. '_a%' = a segunda letra é a.
Na prática O WHERE é o coração da consulta no dia a dia de dados. Combine operadores de comparação com AND/OR/NOT e refine com LIKE, BETWEEN, IN e IS NULL. Próxima ação: teste cada exemplo mentalmente na tabela PESSOAS.
15Funções de agregação e GROUP BYestudado em 26 jun 2026
Conceito Funções de agregação calculam um único resultado a partir de um conjunto de valores (várias linhas). Em vez de ver cada linha, você vê um resumo: total, média, maior, menor.
FunçãoO que faz
COUNTConta linhas. COUNT(*) conta todas; COUNT(DISTINCT col) conta só valores diferentes. Ignora NULL.
SUMSoma os valores de uma coluna.
AVGCalcula a média aritmética.
MINMenor valor.
MAXMaior valor.
Código SQL 
-- Resumo da tabela PESSOAS:
SELECT
  COUNT(*)      AS total,
  AVG(IDADE)  AS media_idade,
  MAX(FILHOS) AS max_filhos
FROM PESSOAS;
-- total=5, media_idade=33, max_filhos=3
AS (alias) AS dá um nome temporário a uma coluna ou tabela, deixando o resultado mais legível. COUNT(*) AS total faz a coluna aparecer como "total".
GROUP BY — agrupar Agrupa as linhas por uma coluna para aplicar a agregação a cada grupo separadamente. Sem GROUP BY, a agregação trata a tabela inteira como um só grupo.
Código SQL 
-- Quantas pessoas têm cada quantidade de filhos:
SELECT FILHOS, COUNT(*) AS qtd
FROM PESSOAS
GROUP BY FILHOS;
-- 0 tem 1, 1 tem 1, 2 tem 2, 3 tem 1
Atenção — COUNT e NULL COUNT(*) conta todas as linhas. COUNT(coluna) conta só as linhas em que a coluna não é NULL. Útil para saber quantos registros têm aquele dado preenchido.
Na prática Agregação + GROUP BY é a base de qualquer relatório: "total por categoria", "média por mês", "contagem por status". Próxima ação: pense numa pergunta do seu trabalho que comece com "quantos" ou "qual a média de".
16ORDER BY e operações aritméticasestudado em 26 jun 2026
ORDER BY — ordenar Define a ordem dos resultados. ASC = crescente (A-Z, 0-9), é o padrão. DESC = decrescente. É a última cláusula do SELECT. Pode ordenar até por colunas não selecionadas e pela posição numérica da coluna.
Código SQL 
-- Do mais velho para o mais novo:
SELECT NOME, IDADE
FROM PESSOAS
ORDER BY IDADE DESC;
-- Marta(48), Miguel(39), Raquel(36), João(22), Eliana(20)
Operações aritméticas Dá para calcular direto no SELECT com + (adição), - (subtração), * (multiplicação), / (divisão) e % (módulo = resto da divisão).
Atenção — divisão por zero Dividir por zero gera erro. Use o WHERE para descartar esses casos antes do cálculo.
Código SQL 
-- Evita divisão por zero filtrando FILHOS > 0:
SELECT NOME, IDADE / FILHOS AS idade_por_filho
FROM PESSOAS
WHERE FILHOS > 0;
-- Eliana (0 filhos) é desconsiderada
Ordem de execução (precedência) Para o cálculo sair certo, o SQL segue esta hierarquia: 1. Parênteses ( ); 2. Expoentes; 3. Multiplicação * e divisão /; 4. Adição + e subtração -. Use parênteses para deixar a intenção explícita.
Na prática ORDER BY organiza a leitura; as operações aritméticas geram métricas direto na consulta. Próxima ação: filtre com WHERE antes de dividir, para nunca cair em divisão por zero.
17Tipos de dados (datatypes)estudado em 26 jun 2026
Conceito Tipos de dados (datatypes) são categorias que definem o tipo de informação que uma coluna pode armazenar. Especificam o formato, o tamanho e a natureza do dado, garantindo armazenamento consistente e eficiente.

TIMESTAMP4 bytesUm ponto específico no tempo.
CHAR vs VARCHAR CHAR é fixo: sempre ocupa o tamanho definido (bom para dados de tamanho constante, como UF ou sexo). VARCHAR é variável: ocupa só o necessário, mais 1 byte (bom para nome, e-mail).
Liga com atributos — UNSIGNED O atributo UNSIGNED remove os negativos e dobra o limite positivo. Ex.: SMALLINT vai de -32.768 a 32.767, mas UNSIGNED vai de 0 a 65.535. Use quando o valor nunca é negativo (idade, quantidade).
Na prática Escolher o tipo certo economiza espaço e evita erros (guardar data como texto atrapalha ordenação e cálculo). Próxima ação: ao criar uma tabela, pergunte para cada coluna: número, texto ou data? E qual o menor tipo que comporta os valores?

FunçãoO que faz
CONCAT / ||Une (concatena) textos. SQLite/Oracle: a || b. MySQL/SQL Server: CONCAT(a,b).

SUBSTRING / SUBSTRExtrai um pedaço (posição inicial + quantidade).

REPLACESubstitui todas as ocorrências de um texto por outro.

Código SQL 
-- Nome em maiúsculas e o tamanho:
SELECT UPPER(NOME) AS maiusculo, LENGTH(NOME) AS tamanho
FROM PESSOAS;

-- Juntando texto com || :
SELECT NOME || ' tem ' || IDADE || ' anos'
FROM PESSOAS WHERE ID=1;
-- Eliana tem 20 anos
Nomes variam por SGBD CONCAT vs ||, LEN vs LENGTH, SUBSTRING vs SUBSTR. O conceito é o mesmo; confira a documentação do seu banco. No app (SQLite) use || e LENGTH.

20Conversão e condicionais (CASE, COALESCE)estudado em 26 jun 2026
Conversão de tipos Transformam um dado de um tipo para outro (texto em número, número em texto, etc.).
FunçãoO que faz
CAST / CONVERTConverte tipos compatíveis. CAST('10' AS INTEGER).
TO_CHARNúmero ou data em texto formatado.
TO_DATETexto em data.
TO_NUMBERTexto em número decimal.
CASE — decisão dentro do SQL Cria lógica condicional na consulta. Sintaxe: CASE WHEN condição THEN resultado [ELSE padrão] END. Economiza código ao evitar tratar condições fora do banco.
Código SQL 
SELECT NOME,
  CASE
    WHEN IDADE >= 40 THEN 'veterano'
    WHEN IDADE >= 30 THEN 'adulto'
    ELSE 'jovem'
  END AS faixa
FROM PESSOAS;
IF / IIF IF(condição, valor_se_verdadeiro, valor_se_falso) faz o mesmo que um CASE simples. No SQLite chama-se IIF. Ex.: IIF(IDADE >= 18, 'maior', 'menor').
COALESCE — primeiro valor não-nulo Percorre uma lista e retorna o primeiro valor que não for NULL. Analogia: ligar para o cliente: tenta o comercial; se vazio, o celular; se vazio, o fixo. COALESCE(comercial, celular, fixo).
NULLIF — NULL se forem iguais Compara dois valores: se forem iguais, retorna NULL; se diferentes, retorna o primeiro. Útil para evitar divisão por zero (NULLIF(divisor, 0)) ou destacar diferenças (meta x realizado).
Código SQL 
SELECT COALESCE(NULL, NULL, 'fixo');  -- fixo
SELECT NULLIF(50, 50);  -- NULL (iguais)
Na prática CASE classifica; COALESCE garante um valor mesmo com campos vazios; NULLIF protege cálculos. São o que deixa a consulta "inteligente" e segura contra nulos.
21JOINs — juntando tabelasestudado em 26 jun 2026
O que é um JOIN JOIN (juntar) combina linhas de duas ou mais tabelas com base em colunas em comum (geralmente PK de uma e FK de outra). É o que permite ver, num só resultado, dados espalhados em tabelas diferentes.
Atenção: em grandes volumes, JOINs têm alto custo computacional e podem deixar a consulta lenta.
Os quatro tipos (diagrama de Venn)
TipoRetorna
INNER JOINSó as linhas que casam nas duas tabelas (a interseção).
LEFT JOINTodas as linhas da tabela da esquerda; o que não casa na direita vem NULL.
RIGHT JOINTodas as linhas da tabela da direita; o que não casa na esquerda vem NULL.
FULL OUTER JOINTodas as linhas dos dois lados; preenche com NULL onde não há par.
Como ler um JOIN FROM A JOIN B ON A.chave = B.chave. O ON diz por qual coluna as tabelas se ligam. Use apelidos (A, B) para encurtar.
Código SQL 
-- Pessoa e produto de cada compra (INNER JOIN):
SELECT P.NOME, PR.NOME
FROM COMPRAS C
JOIN PESSOAS P  ON P.ID = C.ID_PESSOA
JOIN PRODUTOS PR ON PR.ID = C.ID_PRODUTO;

-- Todas as pessoas, mesmo quem não comprou (LEFT JOIN):
SELECT P.NOME, C.ID_PRODUTO
FROM PESSOAS P
LEFT JOIN COMPRAS C ON C.ID_PESSOA = P.ID;
Achar o que NÃO tem par Um LEFT JOIN seguido de WHERE coluna_da_direita IS NULL traz só o que existe apenas na esquerda. Ex.: produtos que nunca foram comprados.
Na prática INNER = interseção; LEFT/RIGHT = um lado inteiro + o que casa; FULL = tudo. Para relatórios reais (cliente + pedidos + produtos) o JOIN é diário.
22UNION — empilhando resultadosestudado em 26 jun 2026
O que é UNION UNION combina (empilha) as linhas de duas ou mais consultas num único resultado. Diferença para o JOIN: o JOIN junta colunas (lado a lado); o UNION junta linhas (uma embaixo da outra).
Analogia Você tem uma lista de compras do mercado e outra da farmácia. O UNION é escrever todos os itens num papel só, um abaixo do outro.
Duas regras obrigatórias 1. Mesmo número de colunas nas duas consultas.
2. Tipos e ordem compatíveis: a coluna na mesma posição deve ter o mesmo tipo (texto com texto, número com número).
ComandoO que faz
UNION ALLJunta tudo sem conferir. Mantém duplicatas. Mais rápido.
UNIONJunta e remove as duplicatas. Resultado limpo, mas exige mais esforço do banco.
Código SQL 
-- Empilha todos os IDs (com repetições):
SELECT ID_PESSOA FROM COMPRAS
UNION ALL
SELECT ID FROM PESSOAS;

-- Só os valores únicos (sem repetir):
SELECT ID_PESSOA FROM COMPRAS
UNION
SELECT ID FROM PESSOAS;
Exemplo visual Lista 1: Verde, Amarelo, Vermelho. Lista 2: Vermelho, Azul, Amarelo.
UNION ALL = 6 itens (Verde, Amarelo, Vermelho, Vermelho, Azul, Amarelo).
UNION = 4 itens únicos (Verde, Amarelo, Vermelho, Azul).
Na prática Use UNION ALL quando quiser tudo (e é mais rápido); use UNION quando precisa de uma lista sem repetições.
23Subqueries (subconsultas)estudado em 26 jun 2026
O que é Uma subquery (subconsulta) é uma consulta dentro de outra. Como uma boneca russa: você faz um SELECT e, dentro dele, coloca outro SELECT.
Dois usos principais Como coluna: traz um valor calculado de outra tabela para cada linha.
Como filtro: limita os resultados com base numa lista gerada por outra consulta (WHERE ... IN (subconsulta)).
Código SQL 
-- Como filtro: só quem fez alguma compra
SELECT NOME FROM PESSOAS
WHERE ID IN (SELECT ID_PESSOA FROM COMPRAS);

-- Como coluna: total de compras de cada pessoa
SELECT NOME,
  (SELECT COUNT(*) FROM COMPRAS WHERE ID_PESSOA=P.ID) AS total
FROM PESSOAS P;
Atenção — desempenho Não há limite para aninhar subconsultas, mas muitas camadas deixam a consulta lenta (o banco tem muito trabalho extra). Use com parcimônia.
Na prática Subconsultas respondem perguntas em duas etapas: "quem comprou?" e depois "traga os nomes desses". Ótimas para filtrar com base em outra tabela.
24Views (visões)estudado em 26 jun 2026
O que é uma View Uma view (visão) é uma tabela virtual: não armazena dados próprios, é apenas um SELECT salvo com um nome. Você a consulta como se fosse uma tabela comum.
Por que usar Reuso: escreve a consulta difícil uma vez e salva.
Segurança: mostra só algumas colunas, escondendo dados sensíveis (ex.: salários).
Simplificação: transforma um código gigante em algo limpo de usar.
CaracterísticaDetalhe
DinâmicaReflete os dados atuais: se a tabela muda, a view mostra a mudança.
PermanenteContinua existindo mesmo após reiniciar o servidor.
LeveNão ocupa espaço extra: é só a instrução salva.
Código SQL 
-- Criar:
CREATE VIEW vw_adultos AS
  SELECT NOME, IDADE FROM PESSOAS WHERE IDADE >= 18;

-- Consultar como se fosse tabela:
SELECT * FROM vw_adultos;
Comandos CREATE VIEW nome AS [SELECT] cria; ALTER VIEW altera a consulta; DROP VIEW apaga a view. DROP VIEW não apaga os dados das tabelas originais, só a visão.
Na prática Views deixam consultas complexas reutilizáveis e seguras (ótimo para esconder colunas sensíveis e padronizar relatórios).
